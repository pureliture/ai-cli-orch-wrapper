# Plan 08-02: Adversarial Commands — `/gemini:adversarial` + `/copilot:adversarial`

**Phase**: 08-adversarial-rescue
**Wave**: 2
**Goal**: Create the two adversarial command files. Each command loads its CLI-specific `adversarial.md` prompt, parses `--focus <value>` from `$ARGUMENTS`, appends a focus-scoped instruction to the prompt, resolves input using the same priority chain as `:review`, and dispatches through `aco_adapter_invoke`.

## Context

Plan 08-01 created the adversarial prompt files and the Wave 0 test stubs. This plan creates the command `.md` files that Claude Code will run when the user invokes `/gemini:adversarial` or `/copilot:adversarial`. The structure is identical to the `review.md` commands (Phase 7) with two additions:
1. `adversarial.md` prompt is loaded instead of `reviewer.md`
2. `--focus` flag is parsed from `$ARGUMENTS` and a focus instruction is appended to the prompt

ADV-04 (`--target` flag) is explicitly deferred — these commands hardcode their adapter key (D-01 pattern, same as review.md).

## Requirements Addressed

- ADV-01: dispatch to adversarial prompt (more aggressive than reviewer.md)
- ADV-02: `--focus security|performance|correctness|all` scopes prompt (default: all)
- ADV-03: input priority = file arg > git diff HEAD > git diff HEAD~1 > "No changes detected"

## Pre-conditions

- Plan 08-01 complete: `.claude/aco/prompts/gemini/adversarial.md` and `.claude/aco/prompts/copilot/adversarial.md` exist
- `.claude/aco/lib/adapter.sh` provides `aco_check_adapter` and `aco_adapter_invoke`
- `.claude/commands/gemini/` and `.claude/commands/copilot/` directories exist (created in Phase 7)

## Tasks

### Task 1: Create `/gemini:adversarial` command

**Action**: Create the file below verbatim. The YAML frontmatter registers the command with Claude Code. The bash block implements ADV-01, ADV-02, and ADV-03 using the exact `$ARGUMENTS` template-substitution pattern established in Phase 7.

Key implementation details:
- `ARGS="${ARGUMENTS:-}"` captures the raw argument string
- Bash regex `[[ "$ARGS" =~ --focus[[:space:]]+([a-z]+) ]]` extracts the focus value into `BASH_REMATCH[1]`
- `ARGS="${ARGS/--focus $FOCUS/}"` + `ARGS="${ARGS## }"` strips `--focus <value>` from ARGS, leaving the optional file path
- Focus instruction is appended to `$ADVERSARIAL_PROMPT` only when FOCUS != "all"
- Input resolution chain is a verbatim copy of review.md's fallback logic (ADV-03)

**File**: `.claude/commands/gemini/adversarial.md`

Exact content:

```markdown
---
name: gemini:adversarial
description: Adversarial code review via Gemini CLI — assumes bugs exist, finds them. Use --focus to scope.
argument-hint: "[--focus security|performance|correctness|all] [path/to/file.ts]"
allowed-tools:
  - Bash
---

Adversarial code review via Gemini CLI. More aggressive than `:review` — starts from the assumption that bugs exist. Optionally scope with `--focus security`, `--focus performance`, `--focus correctness`, or `--focus all` (default). Without a file argument, reviews `git diff HEAD`.

​```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
# ADV-01: Source shared adapter library
source "${SCRIPT_DIR}/../../aco/lib/adapter.sh"

# ADV-01: Check adapter availability
aco_check_adapter "gemini" || exit 1

# ADV-01: Load adversarial prompt (more aggressive than reviewer.md)
ADVERSARIAL_PROMPT_FILE="${SCRIPT_DIR}/../../aco/prompts/gemini/adversarial.md"
if [[ ! -f "$ADVERSARIAL_PROMPT_FILE" ]]; then
  echo "Error: adversarial prompt not found at ${ADVERSARIAL_PROMPT_FILE}" >&2
  echo "Run Plan 08-01 to create it." >&2
  exit 1
fi
ADVERSARIAL_PROMPT=$(cat "$ADVERSARIAL_PROMPT_FILE")

# ADV-02: Parse --focus flag from $ARGUMENTS (template substitution, not argv)
ARGS="${ARGUMENTS:-}"
FOCUS="all"
if [[ "$ARGS" =~ --focus[[:space:]]+([a-z]+) ]]; then
  FOCUS="${BASH_REMATCH[1]}"
  ARGS="${ARGS/--focus $FOCUS/}"
  ARGS="${ARGS#"${ARGS%%[! ]*}"}"
fi

# ADV-02: Validate focus value
case "$FOCUS" in
  security|performance|correctness|all) ;;
  *) echo "Error: invalid --focus '${FOCUS}'. Valid values: security|performance|correctness|all" >&2; exit 1 ;;
esac

# ADV-02: Append focus instruction to prompt (empty string for 'all' = no constraint)
case "$FOCUS" in
  security)
    FOCUS_INSTR="FOCUS CONSTRAINT: Concentrate exclusively on security vulnerabilities. Treat every unvalidated input, every auth check, every secret access, and every dependency as a potential attack surface. Other findings are out of scope for this review."
    ;;
  performance)
    FOCUS_INSTR="FOCUS CONSTRAINT: Concentrate exclusively on performance issues. Identify algorithmic complexity problems, N+1 query patterns, unbounded memory growth, unnecessary I/O, and blocking operations. Other findings are out of scope."
    ;;
  correctness)
    FOCUS_INSTR="FOCUS CONSTRAINT: Concentrate exclusively on correctness and logic errors. Identify wrong output, missed edge cases, off-by-one errors, type coercion surprises, and race conditions. Other findings are out of scope."
    ;;
  all)
    FOCUS_INSTR=""
    ;;
esac
if [[ -n "$FOCUS_INSTR" ]]; then
  ADVERSARIAL_PROMPT="${ADVERSARIAL_PROMPT}"$'\n\n'"${FOCUS_INSTR}"
fi

# ADV-03: Input resolution — identical priority chain to review.md
FILE_ARG="$ARGS"
if [[ -n "$FILE_ARG" ]]; then
  if [[ ! -f "$FILE_ARG" ]]; then
    echo "Error: file not found: $FILE_ARG" >&2
    exit 1
  fi
  CONTENT=$(cat "$FILE_ARG")
else
  CONTENT=$(git diff HEAD 2>/dev/null || true)
  if [[ -z "$CONTENT" ]]; then
    CONTENT=$(git diff HEAD~1 2>/dev/null || true)
  fi
  if [[ -z "$CONTENT" ]]; then
    echo "No changes detected"
    exit 0
  fi
fi

# Dispatch through adapter (output verbatim)
aco_adapter_invoke "gemini" "$ADVERSARIAL_PROMPT" "$CONTENT"
