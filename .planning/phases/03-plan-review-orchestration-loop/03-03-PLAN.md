CAO server endpoints confirmed on this machine:
```text
GET  /health
POST /sessions?provider=<provider>&agent_profile=<agent>&session_name=<name>&working_directory=<cwd>
GET  /terminals/{terminal_id}
POST /terminals/{terminal_id}/input?message=<prompt>
GET  /terminals/{terminal_id}/output?mode=last
POST /terminals/{terminal_id}/exit
```

Relevant response shapes from CAO OpenAPI:
```ts
interface CaoTerminal {
  id: string;
  name: string;
  provider: string;
  session_name: string;
  agent_profile?: string | null;
  status?: 'idle' | 'processing' | 'completed' | 'waiting_user_answer' | 'error' | null;
}

interface TerminalOutputResponse {
  output: string;
  mode: 'full' | 'last';
}
```

Locked decisions to enforce here:
- D-12/D-13: only `review.status.json.status` controls approval
- D-14: `approved` stops the loop
- D-15/D-16: max iteration ends non-approved and preserves artifacts
- D-17: surface clear next action after non-approved exit
- research architecture: use fresh planner/reviewer sessions per iteration
```

   export interface CaoTerminal {
     id: string;
     name: string;
     provider: string;
     session_name: string;
     agent_profile?: string | null;
     status?: 'idle' | 'processing' | 'completed' | 'waiting_user_answer' | 'error' | null;
   }

   export interface TerminalOutput {
     output: string;
     mode: 'full' | 'last';
   }

   export class CaoHttpClient {
     constructor(baseUrl?: string);
     checkHealth(): Promise<void>;
     createSession(input: {

       agentProfile: string;
       sessionName: string;
       workingDirectory: string;
       launchArgs?: string[];
     }): Promise<CaoTerminal>;
     sendInput(terminalId: string, message: string): Promise<void>;
     getTerminal(terminalId: string): Promise<CaoTerminal>;
     waitForCompletion(
       terminalId: string,
       options?: { pollIntervalMs?: number; timeoutMs?: number },
     ): Promise<CaoTerminal>;
     getOutput(terminalId: string, mode?: 'full' | 'last'): Promise<TerminalOutput>;
     exitTerminal(terminalId: string): Promise<void>;
   }
   ```
   - default `baseUrl` to `http://localhost:9889`
   - use `fetch` + `URL` + `URLSearchParams`
   - `checkHealth()` must throw `CAO server is not running. Start it with 'cao-server'.` when unavailable
   - `waitForCompletion()` must poll `GET /terminals/{id}` until:
     - `completed` → success
     - `error` → throw
     - `waiting_user_answer` → throw, because the automated loop cannot continue reliably
   - keep `exitTerminal()` best-effort; do not let cleanup failure hide a real workflow result

   ```ts
   export interface WorkflowRunResult {
     exitCode: 0 | 1 | 2;
     runDir: string;
     finalStatus: 'approved' | 'max_iterations' | 'failed';
     iterationCount: number;
   }

   export async function runWorkflow(
     workflow: ResolvedWorkflowDefinition,
     options?: {
       repoRoot?: string;
       client?: Pick<

         'checkHealth' | 'createSession' | 'sendInput' | 'waitForCompletion' | 'getOutput' | 'exitTerminal'

       pollIntervalMs?: number;

     },
   ): Promise<WorkflowRunResult>;
   ```
   Implement the loop exactly:

   - do not parse review prose to decide approval

Record enough metadata in `iteration.json` and `state.json` to prove what happened: iteration number, terminal/session ids, timestamps, final review status.
