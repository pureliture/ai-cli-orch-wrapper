## Overview

v1.2 pivots the project from a TypeScript CLI binary to a pure Markdown+Bash slash-command bridge layer. The entire `src/` directory was removed. The output is 14 slash commands (7 per CLI: gemini + copilot) that let Claude Code orchestrate Gemini CLI and Copilot CLI as subagents — no external binary required.

**Core deliverables:**
- `adapter.sh` — shared bash library (10 functions) that all commands source
- `.wrapper.json` v2.0 — routing schema + adapter availability detection
- 14 slash commands across `/gemini:*` and `/copilot:*` namespaces
- Background task system for async review/adversarial dispatch
- 44-test automated suite (0 failures)

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| v1.2 pivot: CC slash commands instead of TypeScript CLI | ccg-workflow/codex-plugin-cc pattern fits purpose better; no binary needed | ✓ Delivered |
| `src/` deleted | TypeScript CLI code incompatible with Markdown+Bash output | ✓ Done |
| Per-CLI namespace (`/gemini:*`, `/copilot:*`) instead of routing config | Explicit over implicit; no routing ambiguity | ✓ Delivered |
