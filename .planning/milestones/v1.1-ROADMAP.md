# Milestone v1.1: Wrapper Command Consolidation

**Status:** SHIPPED 2026-03-31
**Phases:** 04-05
**Total Plans:** 7
**Total Tasks:** 6

## Overview

v1.1 narrowed the wrapper back to one public CLI name and one stable repo-local runtime contract. The milestone locked `aco` as the only supported user-facing command, added real-machine recovery for stale `wrapper` shims, and preserved `.wrapper.json` plus `.wrapper/workflows` so existing repositories kept working while internal naming moved to Aco branding.

## Phases

### Phase 04: Canonical Command Surface

**Goal**: Users can discover, invoke, and recover to the `aco` command without ambiguity while `.wrapper*` runtime paths stay intact for the next phase.
**Depends on**: Phase 03
**Plans**: 3 plans

Plans:

- [x] 04-01 — Cut over the install/help/version surface and public quick-start guidance to `aco`
- [x] 04-02 — Fail fast on stale `wrapper` entrypaths and align setup-managed wording to `aco`
- [x] 04-03 — Remove stale package-owned `wrapper` shims during install/relink and verify the real machine surface is `aco`-only

**Details:**

- `package.json`, help output, version output, and ordinary unknown-command guidance were unified around `aco`
- stale `wrapper` entrypaths now fail fast to `Use aco ...` instead of behaving like supported commands
- relink/install cleanup removes only package-owned legacy `wrapper` shims and leaves unrelated executables untouched

### Phase 05: Wrapper Runtime Contract

**Goal**: Users can trust the existing v1.0 runtime flows to keep working once `aco` is the canonical public command while the `.wrapper*` repo-local contract remains in place.
**Depends on**: Phase 04
**Plans**: 4 plans

Plans:

- [x] 05-01 — Cleanup legacy lockfile references and finalize phase metadata
- [x] 05-02 — Rename internal Config symbols and module to Aco branding
- [x] 05-03 — Rename internal Artifact/Orchestration symbols to Aco branding
- [x] 05-04 — Protect built-ins from alias overrides and finish runtime branding alignment

**Details:**

- internal config and artifact symbols moved to Aco naming while `.wrapper.json` and `.wrapper/workflows` stayed stable on disk
- `aco setup` kept generating and preserving `.wrapper.json` as the repo-local contract
- built-in commands now stay ahead of aliases, and reserved alias names are hidden from help output instead of blocking dispatch
- workflow execution still lands artifacts under `.wrapper/workflows` while preferring `ACO_CAO_BASE_URL` with fallback to `WRAPPER_CAO_BASE_URL`

## Milestone Summary

**Key Accomplishments**

- `aco` is now the sole supported public CLI surface
- stale global `wrapper` shims are cleaned up safely during relink/reinstall
- `.wrapper.json` and `.wrapper/workflows` remained stable through the internal Aco branding transition
- built-ins-first dispatch is locked again for `help`, `setup`, `workflow`, and `workflow-run`
- targeted runtime coverage now verifies the command surface and repo-local runtime contract together

**Key Decisions**

- keep the public CLI rename (`aco`) decoupled from the on-disk repo-local contract (`.wrapper*`)
- treat reserved alias names as inert so built-ins stay authoritative after the command-surface cutover
- prefer `ACO_CAO_BASE_URL` while retaining `WRAPPER_CAO_BASE_URL` as a compatibility fallback

**Issues Resolved**

- older machines could retain a stale global `wrapper` shim even after the public bin moved to `aco`
- late milestone audit found `CMD-03` had regressed; built-ins-first dispatch was restored before archive

**Issues Deferred**

- documentation and architecture cleanup remain intentionally deferred to v1.2
- isolated workspaces and richer workflow ergonomics remain intentionally deferred to v1.3

**Technical Debt**

- `04-VALIDATION.md` and `05-VALIDATION.md` remain draft/partial rather than fully closed validation artifacts
- there is still no explicit automated test proving `ACO_CAO_BASE_URL` wins when both env vars are set
